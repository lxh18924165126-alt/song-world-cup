# 视觉风格与组件规范

## 视觉方向
- 暗黑竞技转播风；
- 背景深黑、炭灰；
- 主强调为竞技红；
- 冠军与奖励使用金色；
- 依靠聚光灯、奖杯、对阵线与信息层级建立仪式感；
- `docs/00_overview/project_summary.md` 规定不使用单曲封面；视觉层以排版、聚光灯、奖杯与对阵线建立辨识度。

## 设计令牌
- 背景：`#090B0E / #11161C / #171D24`
- 强调红：`#E53935`
- 强调金：`#F5C46B`
- 辅助蓝：`#4DA3FF`
- 主文字：`#F5F7FA`
- 次文字：`#AAB4C0`
- 常用圆角：`16 / 20 / 24`

## 关键组件
- 歌曲卡片：歌名、歌手、媒体按钮、选中态、禁用态；
- 对决行：左卡片 / VS / 右卡片；
- 轮次头部：赛事名称、赛区、轮次、进度；
- 固定签表画布：完整世界坐标、前一 / 当前 / 后一共三个视窗的虚拟渲染、局部晋级连线和只由系统控制的镜头；
- 画布导航：上一组、下一未完成组、当前轮进度滑杆和晋级汇总锁轮节点；
- 抽签预览卡：赛制摘要、轮空数、签表缩略图；
- 结果摘要卡：紧接页头的紧凑完赛标题、冠军主卡和默认折叠的亚军 / 四强详情；
- 我的赛事列表项：状态标签、信息、操作按钮；
- 后台统计卡：指标值、健康状态、快捷操作。

## 比赛画布动效

- 普通轮次使用 560ms 缓动平移，选胜后先点亮通往晋级节点的路径，再在约 700ms 后移动；
- 移动端普通轮次保持每屏 4 场：比赛节点约 220px 宽，每首歌曲使用独立边框卡片和至少 44px 点击区，两个歌曲卡之间以 `VS` 明确分隔；同屏两个约 120px 宽的“下一轮”节点保持晋级关系。首场距画布顶部保持 16–24px，极矮画布降为每屏 2 场；后续普通轮次及八强对当前视窗应用 128px 行距的局部投影，八强改为纵向居中、普通轮次保持顶部锚定，完整签表的稳定世界坐标不变，避免倍增世界行距把同组节点裁出画布；
- 宽屏 8 场模式使用紧凑节点与 72px 行距，必须同时完整露出首尾节点；可用画布高度不足时自动降为 6 / 4 / 2 场，节点不得被 HUD 或底部导航裁切；
- 晋级连线从当前对阵框外沿连接到下一轮节点外沿，不得穿过歌曲卡片；半决赛聚光与“决赛席位之争”以当前画面中的总决赛晋级节点为视觉轴线，并随左右赛区镜头同步移动；
- 八强、半决赛、总决赛依次增强聚光、红金描边、路径流光和金色粒子；总决赛额外显示 45° 皇冠角标、半透明奖杯背光，并在画布两侧保留半露且置灰的上一轮半决赛卡片及其汇入连线，不使用远程图片或持续 JavaScript 粒子循环；
- `prefers-reduced-motion: reduce` 时取消大幅平移与粒子，只保留即时定位和状态高亮；低性能设备同样禁用粒子、滤镜和连线循环动画。

## 对阵图导出视觉

- 导出 PNG 复用比赛固定签表的镜像世界坐标和暗黑竞技转播语言，而不是另行使用线性简化签表；
- 静态图片保留深色网格、双侧聚光、红金节点、冠军中心舞台、轻量粒子，以及用发光虚线定格的胜者晋级流光；
- 图片头部展示赛事名称、签表规模和清晰度，右侧二维码必须有白色静区，并标注其指向公开只读赛果；
- 512~4096 强签表允许按画布上限压缩节点和隐藏过小文字，但签位、晋级连线和胜者路径必须保持完整。
