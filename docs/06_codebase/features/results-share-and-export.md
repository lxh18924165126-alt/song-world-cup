# 完赛结果、赛后分享与本地导出

## 功能范围与用户入口

- 决赛锁定后，比赛页自动进入 `/t/:id/result` 创建者结果页，首屏突出冠军；亚军和并列四强默认折叠在“完整赛果”入口中。
- 结果页优先展示 IndexedDB 中已生成的本地赛果；云端补传和分享状态读取在后台进行，不阻塞赛果展示。
- 赛后分享默认关闭；创建者主动开放或下载带二维码的对阵图时返回 `/share/:token` 只读链接，并可关闭或重置。下载操作静默开放分享，不弹出额外确认。
- 分享链接复制优先使用 Clipboard API，并在 IP + HTTP 等非安全上下文中使用浏览器选区复制回退。
- 重置会立即使旧链接永久失效；关闭后当前链接也不可访问。
- 创建者可在设备本地生成复用比赛超大画布视觉、附公开赛果二维码的对阵图 PNG；不提供结果海报生成入口。

## 代码入口与职责

- `packages/domain/src/tournament.ts`：从已锁定签表派生冠军、亚军、四强、对决数和冠军胜场。
- `apps/api/src/shares.ts`：完赛校验、分享状态、开放、关闭、重置和公共只读载荷。
- `apps/api/migrations/0005_tournament_shares.sql`：分享令牌、开放状态和更新时间。
- `apps/web/src/features/results/ResultPage.tsx`：创建者结果页、本地优先展示、云端同步失败重试、分享控制和导出入口。
- `apps/web/src/features/results/PublicSharePage.tsx`：无媒体按钮、默认 `noindex` 的只读赛果页。
- `apps/web/src/features/results/ResultSummary.tsx`：创建者页和只读页共享的赛果展示。
- `apps/web/src/features/results/export.ts`：复用固定签表世界坐标与连线几何，在 Canvas 本地绘制网格、聚光、红金节点、流光、粒子、冠军舞台和二维码，并下载 PNG。
- `apps/web/src/features/tournament/bracketCanvas.ts`：比赛画布与导出共同使用的稳定镜像节点和晋级连线几何。

## 调用、数据与持久化

1. 分享 API 必须携带赛事恢复令牌，并拒绝尚未完成的赛事。
2. 首次开放创建随机分享令牌；关闭只改变开放状态；重置原子替换令牌并重新开放。
3. 公共接口只按开放令牌读取已完赛赛事、快照标题、封面和结果歌曲，不返回原音乐平台歌单地址。
4. 下载时若分享未开放，结果页先调用现有 `open-share` 接口并更新页面分享状态；取得有效公开地址后在本地生成二维码，再使用 `canvas.toBlob` 直接下载，不上传图片。
5. 通用对阵图画布限制为 8192 像素高；桌面原始超清版最多 32760 像素高，以适配 512~4096 强签表与浏览器画布上限。
6. 页面先从 IndexedDB 读取完赛进度并结束“正在整理”状态；若仍有待同步事件，后台通过批量事件接口补传，成功后再读取分享状态。

## 状态、不变量与失败处理

- 未完赛赛事不能开放、重置或读取公开分享。
- 默认没有分享记录；主动开放或下载带二维码的对阵图时才创建链接。
- 公共分享页不渲染任何试听、播放或来源音乐平台跳转按钮，并设置 `noindex,nofollow`。
- 对阵图始终使用代码绘制的超大画布暗黑竞技视觉，不依赖远程图片或云端渲染；动态流光在静态 PNG 中以发光虚线定格。
- 云端同步中、离线或失败时继续保留本地赛果，并提供明确状态与“重试云端同步”入口；分享按钮和带公开二维码的对阵图下载在云端确认完赛前禁用，避免生成无法访问的二维码。

## 验证路径

- 领域测试覆盖冠军、亚军、四强、实际对决数和冠军胜场统计。
- Web 导出布局测试覆盖 4 强最低尺寸和 4096 强标准/原始高度上限。
- 本地集成实测覆盖开赛至完赛、默认关闭、开放访问、重置旧链接 404、关闭新链接 404。
- 移动端 Playwright 回归会挂起完赛同步并注入 502，验证挂起期间赛果已经显示、失败后本地赛果保留、重试成功后下载 PNG 会静默开放分享，并可继续关闭分享。
- `pnpm test`、`pnpm typecheck` 与 `pnpm build` 验证领域、React、超大画布对阵图导出和 Worker 迁移绑定。
