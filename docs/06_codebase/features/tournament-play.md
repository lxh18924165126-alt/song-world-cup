# 正式开赛与逐轮比赛

## 功能范围与用户入口

- 抽签预览页调用 `POST /api/drafts/:id/start` 锁定当前歌曲集合、签位与路径，并进入 `/t/:id/play#token=...`。
- 比赛页按当前轮次和赛区展示真实对阵；手机每页 4 场，桌面每页 8 场，八强每页 2 场。
- 半决赛和总决赛自动切换到 `/t/:id/final` 专属舞台，每页一场；总决赛点击歌曲后立即依次落地选择与锁轮事件。
- `/t/:id/bracket` 从当前缓存或云端恢复完整轮次，以横向签表展示待选、已晋级和冠军状态。
- 点击未选歌曲设为胜者，再点已选歌曲取消，点击对手直接改选；所有有效对决完成后才能锁轮。
- 锁轮固定当前轮胜者并生成下一轮；锁定决赛后写入冠军并把赛事标记为完成。
- 歌曲选择按钮与媒体按钮分离；媒体按钮短按打开快照保存的来源平台歌曲链接，有 `previewUrl` 时长按 500ms 播放并在 30 秒内自动停止。只读分享页不复用该按钮。

## 代码入口与职责

- `apps/web/src/features/tournament/PlayPage.tsx`：分页对决、八强重点状态、半决赛/总决赛专属舞台、即时决赛锁定、进度与冲突处理。
- `apps/web/src/features/tournament/BracketPage.tsx`：进行中及完赛完整对阵总览。
- `apps/web/src/features/tournament/api.ts`：开赛、恢复、选择和锁轮客户端。
- `apps/api/src/tournaments.ts`：赛事授权、D1 条件更新、版本冲突和状态机调用。
- `apps/api/src/index.ts`：赛事路由及 400、404、409 错误映射。
- `packages/domain/src/tournament.ts`：从签表创建首轮、选择/取消/改选、锁轮、生成下一轮和决赛完结。

## 调用、数据与持久化

1. 正式开赛复用草稿恢复令牌做创建者凭证；D1 的 `tournaments` 仍只保存 SHA-256 哈希。
2. `draft_id` 唯一约束使重复开赛请求返回同一个赛事，避免双击创建两份进度。
3. 首轮轮空保持 `auto_bye` 并带自动胜者，不进入比赛操作页；其余比赛从 `pending` 进入 `picked`，锁轮后变为 `locked`。
4. 每次选择、取消、改选或锁轮都提交当前版本；更新语句使用 `WHERE version = ?`，成功后版本加一，旧页面返回 409。
5. 当前进度以 `progress_json` 原子保存，包含所有已生成轮次、对阵、胜者、当前轮和冠军；`status`、版本及完成时间同时作为可索引列保存。

迁移 `apps/api/migrations/0003_tournaments.sql` 创建 `tournaments`，并通过草稿和快照外键保持来源一致。

## 状态、不变量与失败处理

- 已开赛草稿不能重抽；已锁轮次不能修改；轮空结果不能修改。
- 胜者必须是当前对阵中的一个条目；取消选择把比赛恢复为 `pending`。
- 当前轮任一有效对决没有胜者时拒绝锁轮；下一轮只从已锁定胜者生成。
- 赛事完成后状态机拒绝继续选择或锁轮。
- 赛事页面已接入 IndexedDB 比赛缓存、离线事件队列、单设备编辑租约、保护期到期接管，以及云端覆盖或本地分支另存的显式冲突处理。
- 锁轮后的下一轮先原子写入本地缓存；进入半决赛专属路由时立即呈现该缓存，编辑租约续期与待同步事件在后台完成，慢网络不会把页面阻塞在恢复提示上。

## 验证路径

- `pnpm test` 覆盖开赛轮空、选择/取消/改选、未完成拒绝锁轮、下一轮生成和决赛冠军。
- `pnpm typecheck` 与 `pnpm build` 检查领域、Worker 和 React 页面契约。
- 本地 API 实测包含开赛 201、开赛后重抽 409、选择 200、旧版本选择 409，以及完成 32 场后锁轮进入下一轮。
- 浏览器入口为抽签预览的“正式开始”按钮和恢复链接；Playwright 已从 16 强完整跑到结果页，验证手机 4 场、八强 2 场、左右半决赛、总决赛即时锁定、自动翻页、离线选择和第二设备只读，并挂起半决赛路由的编辑权心跳，确认本地下一轮仍可立即显示。
