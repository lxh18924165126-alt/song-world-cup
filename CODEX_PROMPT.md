# 二、技术栈与硬约束
- pnpm monorepo
- React + TypeScript + Vite
- Cloudflare Workers + D1 + Durable Objects + Queues
- IndexedDB + Service Worker
- Vitest + Playwright
- 只能使用 Cloudflare 免费套餐
- 导出图必须本地生成，不得依赖云端渲染
- 若本机代理影响 Cloudflare 联调或部署，则把远程步骤放最后处理

# 三、必须实现的功能
1. QQ 音乐公开歌单导入（服务端优先，浏览器直连为自动备用）
2. 歌曲检查页：搜索、排除、批量选择、保留重复歌曲
3. 赛事设置：名称、规模、所有歌曲随机
4. 抽签预览：无限重新抽签，云端草稿与恢复链接
5. 正式比赛：
   - 手机端一屏 4 场
   - 点击选择 / 取消 / 改选
   - 自动切页
   - 左赛区先于右赛区
   - 轮空自动晋级
   - 八强开始重点舞台
   - 半决赛与总决赛专属页面
6. 离线继续、刷新后继续、编辑权保护期、接管与冲突分支另存
7. 本地“我的赛事”页面
8. 完整结果页
9. 创建者主动开放赛后分享后才生成只读分享链接
10. 竖版结果海报 PNG 与纯对阵图 PNG 本地导出
11. 微信 / QQ 登录完整代码与模拟 Provider
12. 登录后赛事迁移选择页
13. 最小化运营后台
14. 最终部署到 `workers.dev`，返回实际访问地址

# 四、开发方法
- 不要一次性堆大量代码。
- 按功能切片推进，每完成一个切片就运行验证。
- 前端不能只做静态页面，必须带真实交互、状态管理和离线逻辑。
- 后端不能只留空壳，必须包含真实数据模型、迁移与 API。
- 效果图必须尽量 1:1 复刻，不得自由发挥成另一套设计。
- 若某项能力因外部平台凭证缺失无法生产验收，需要明确标记“代码已完成，生产联调待凭证”。
